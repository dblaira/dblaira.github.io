# Browser Checklist

Use this when executing the reverse-spec workflow in the browser.

## Order

1. Start from an existing visual artifact if one exists
2. Build a composition board before explaining
3. Make selection visual and direct
4. Extract notes from the chosen visual
5. Only then move into components and implementation

## Composition Board Checklist

- [ ] Use real reference imagery, finished HTML, or screenshots
- [ ] Show 2-6 meaningful composition options
- [ ] Give each option a short caption
- [ ] Keep controls minimal and deferential
- [ ] Make the image area larger than the text area
- [ ] Avoid developer-looking panels and wireframe chrome

## Selection Mechanics

- [ ] Let the user click, crop, circle, reject, or cluster
- [ ] Treat captions as support, not the main medium
- [ ] Prefer side-by-side comparison when the question is comparative
- [ ] Update the right-side note region from the chosen option

## Reverse-Spec Notes

Extract these fields from the selected visual:

- pattern
- foreground/background
- center of gravity
- scale relationships
- implied components
- implied interaction model
- what should remain silent

## Red Flags

Stop and restart if you are about to:

- explain before showing
- use Mermaid or boxes by default
- make all concepts the same size
- turn the board into a code editor
- treat text as clearer than images for a visual task
